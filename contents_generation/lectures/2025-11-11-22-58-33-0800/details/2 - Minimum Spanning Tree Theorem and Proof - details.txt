# 2. Minimum Spanning Tree Theorem and Proof

This lecture introduces the Minimum Spanning Tree (MST) problem and presents a fundamental theorem that guarantees the optimality of certain edges in an MST. The core idea is that for any way a graph's vertices are divided into two sets, the minimum-weight edge connecting these sets must be part of every MST. This theorem is then rigorously proven using a proof by contradiction, highlighting its importance for developing MST algorithms.

## üìö Understanding the Minimum Spanning Tree (MST) Problem
The **Minimum Spanning Tree (MST)** problem involves finding a connected subgraph that spans all vertices of a given graph with the minimum possible total edge weight. This problem has practical applications, such as determining how to connect a set of network nodes with minimum traffic or congestion. Many other problems can also be formulated and solved using MST algorithms. ‚ü¶s000046, s000049, s000050, s000051‚üß

## üí° The Unique Weight Assumption
For the purpose of discussing MST algorithms and their proofs, it is often assumed that all edge weights in the graph are *unique*. This assumption simplifies the algorithms and proofs significantly. ‚ü¶s000062, s000067‚üß If, in a real-world scenario, weights are not unique (e.g., two edges have a weight of '2'), a common trick is to slightly perturb one of the weights by adding a very small number, *epsilon*, making it '2 + epsilon'. This effectively makes all weights unique without altering the overall structure or solution significantly. ‚ü¶s000070, s000071, s000072‚üß

## üå≥ Essential Tree Properties
A **tree structure** is a type of graph that, by definition, contains no cycles. For a graph with *N* vertices, a tree will always have exactly *N-1* edges, no more and no less. ‚ü¶s000077, s000078, s000079, s000080, s000082‚üß This *N-1* edge count represents the minimum number of edges required for a graph to be connected. If a graph has fewer than *N-1* edges, it cannot be connected. ‚ü¶s000083, s000084, s000318, s000320‚üß
Adding just one edge to an existing tree structure will inevitably create a cycle within the graph, transforming it from a tree into a graph with a cycle. Conversely, if an edge is removed from such a cycle, the graph can revert to a tree structure without cycles, provided the removed edge was part of the cycle. ‚ü¶s000089, s000090, s000091, s000101, s000103, s000106, s000107, s000108‚üß

## ‚úÇÔ∏è Graph Partitioning and the `E_min` Edge
Consider an original graph *G* with *N* vertices and *E* edges, where edges have associated weights. This graph can be **partitioned** into two non-empty sets of vertices, let's call them *V1* (left) and *V2* (right). ‚ü¶s000110, s000112, s000113, s000116, s000127, s000128‚üß The specific way the graph is partitioned does not matter; it can be any valid bipartition where both sets are non-empty. ‚ü¶s000115, s000120, s000121, s000127‚üß
Since the original graph is assumed to be connected, there must be at least one edge connecting a vertex in *V1* to a vertex in *V2*. ‚ü¶s000132, s000133, s000135, s000138‚üß Among all such edges that cross this partition (from *V1* to *V2*), we identify the one with the *minimum weight*. This specific edge is denoted as **`E_min`**. ‚ü¶s000144, s000147, s000150, s000202‚üß

## üìú Statement of the MST Theorem
The **Minimum Spanning Tree (MST) Theorem** states that for *every possible partition* of the graph's vertices into two non-empty sets, the edge with the minimum weight (`E_min`) that connects these two sets *must be included in every Minimum Spanning Tree* of the graph. This is a powerful and general statement, applying to any partition, not just a specific one. ‚ü¶s000151, s000194, s000195, s000198, s000205, s000206, s000292, s000293‚üß

## üõ°Ô∏è Proof of the MST Theorem by Contradiction

The MST Theorem can be proven using a **proof by contradiction**.

### ‚û°Ô∏è Setting up the Contradiction
Assume, contrary to the theorem, that there *exists* an MST, let's call it *T'*, which *does not* contain the edge `E_min` for a given partition. ‚ü¶s000208, s000210, s000221, s000243, s000247‚üß Since *T'* is a spanning tree, it must connect all vertices, meaning it must have at least one edge crossing the partition from *V1* to *V2*. If it didn't, it would be a forest (disconnected components), not a spanning tree. ‚ü¶s000225, s000228, s000229, s000230‚üß

### ‚û°Ô∏è Forming a Cycle and Identifying `E_cycle`
Now, consider adding the edge `E_min` to our assumed MST *T'*. Since *T'* is already a tree, adding any new edge will create exactly one cycle. ‚ü¶s000248, s000256, s000257, s000258, s000259, s000262, s000263‚üß This newly formed cycle must include `E_min`. Because `E_min` connects *V1* and *V2*, the cycle must also contain at least one other edge that crosses the partition from *V1* to *V2*. Let's call this other edge **`E_cycle`**. ‚ü¶s000265, s000266, s000270‚üß

### ‚û°Ô∏è Constructing a Lighter Spanning Tree
Next, we remove `E_cycle` from the graph (*T'* + `E_min`). The resulting graph, let's call it *T''*, will still be a spanning tree because we added one edge (`E_min`) and removed another (`E_cycle`) from a structure that was originally a tree plus one edge. ‚ü¶s000270, s000272, s000273, s000275, s000276‚üß
Now, let's compare the total weights:
*   The weight of *T''* is `Weight(T') + Weight(E_min) - Weight(E_cycle)`. ‚ü¶s000280, s000283‚üß
By definition, `E_min` was chosen as the *minimum weight edge* crossing the partition. Since `E_cycle` also crosses the same partition, it must be that `Weight(E_min) < Weight(E_cycle)`. ‚ü¶s000280, s000284, s000285, s000286‚üß

### ‚û°Ô∏è The Contradiction and Conclusion
Because `Weight(E_min)` is strictly less than `Weight(E_cycle)`, it follows that `Weight(T'')` must be strictly less than `Weight(T')`. ‚ü¶s000280, s000287‚üß This creates a contradiction: we assumed *T'* was an MST (meaning it had the minimum possible total weight), but we have just constructed *T''*, which is also a spanning tree and has an even lower total weight. Therefore, our initial assumption that an MST *T'* could exist without `E_min` must be false. ‚ü¶s000288, s000289, s000290‚üß This proves that `E_min` must indeed be part of every MST. ‚ü¶s000292, s000293‚üß

## ‚ö†Ô∏è Importance of Unique Edge Weights
It is crucial to note that this proof relies on the assumption of *unique edge weights*. If weights are not unique, then `Weight(E_min)` might be *equal to* `Weight(E_cycle)` (i.e., `Weight(E_min) <= Weight(E_cycle)`). In such a case, replacing `E_cycle` with `E_min` would result in a new spanning tree *T''* with a weight *equal to* or less than *T'*, not strictly less. ‚ü¶s000294, s000295, s000296, s000297‚üß This means the proof by contradiction, as presented, would not directly hold because it wouldn't necessarily lead to a *strictly* lighter tree, thus not contradicting the "minimum" aspect of MST. However, this is considered a minor issue that can be addressed. ‚ü¶s000298, s000299, s000300, s000301‚üß

## ‚öôÔ∏è Algorithmic Implications of the Theorem
The MST Theorem provides a fundamental strategy for designing MST algorithms. The general game plan is to iteratively identify an `E_min` edge for a chosen partition and add it to the solution. This process is repeated, creating new partitions and finding new `E_min` edges, until *N-1* edges have been selected, forming a complete MST. ‚ü¶s000302, s000303, s000304, s000306, s000307, s000308, s000309, s000313, s000314, s000317, s000318, s000320‚üß Different methodologies for creating these partitions lead to various MST algorithms, such as Prim's and Kruskal's algorithms. ‚ü¶s000104, s000321‚üß
